# Learning Pathway: Web Developer — Beginner to Professional Workflow

> **Who this is for:** Developers who have completed **Front-End Basics (web)** and can build static, interactive pages but feel unsure working in a real team or professional environment.
> **Goal:** Work and ship code like a professional front-end developer.
> **Estimated Time:** ~45 hours
> **Prerequisite:** [Front-End Basics (web)](../../Courses/webdev/index.md)

---

## Pathway Overview

This pathway moves you through the Front-End Intermediate course in a single opinionated sequence. Each unit introduces a skill that is immediately used in the units that follow.

| Step | Content | Type | Skills Gained |
| :--- | :--- | :--- | :--- |
| 1 | [Front-End Intermediate (web)](../../Courses/webdev_intermediate/index.md) | Course | Git, CLI, async JS, ES6+, Browser APIs, CSS animations, DevTools |
| 2 | [Intermediate Capstone Project](../../Projects/webdev_intermediate_capstone/index.md) | Project | Apply all units to a real API-powered, animated, version-controlled deliverable |

## Milestones

**Milestone 1 — "I can work with Git and the terminal"**
After Unit 1 (Git & CLI), you should be able to initialise a repo, commit changes, create branches, open a pull request, and deploy via GitHub Pages — without touching a GUI.

**Milestone 2 — "I understand how JavaScript handles time"**
After Unit 2 (Async JS), you should be able to explain the event loop, write Promises and async/await from memory, and fetch data from a public REST API and render it in the DOM.

**Milestone 3 — "I write modern JavaScript"**
After Unit 3 (ES6+), your code should use destructuring, modules, optional chaining, and expressive array methods as default habits — not things you have to look up.

**Milestone 4 — "I can persist state and navigate programmatically"**
After Unit 4 (Browser APIs), you should be able to use `localStorage` to persist user preferences, manipulate the URL with the History API, and lazy-load content with the Intersection Observer.

**Milestone 5 — "I can bring UI to life"**
After Unit 5 (CSS Transitions & Animations), you should be able to animate elements with `transition`, `transform`, and `@keyframes`, and know when *not* to animate for accessibility reasons.

**Milestone 6 — "I can debug anything"**
After Unit 6 (DevTools), you should be able to inspect the DOM, read the Network tab, set breakpoints in the Sources panel, and profile page performance — without guessing.

## Recommended Tools & Environment

Before starting, ensure you have:

- [Git](https://git-scm.com/) installed and configured (`git config --global user.name/email`)
- [Visual Studio Code](https://code.visualstudio.com/) — use the integrated terminal throughout
- A [GitHub](https://github.com) account — you will push real code here
- A modern browser (Chrome or Firefox) — DevTools is opened constantly
- A free [Netlify](https://netlify.com) or [GitHub Pages](https://pages.github.com) account for the capstone deployment

## What Comes Next

After completing this pathway you will be ready for:

- **React / Component-based UI** — you will have the JS fundamentals (modules, async, ES6+) to learn a modern framework
- **Node.js & Back-End basics** — the terminal and Git skills from Unit 1 transfer directly
- **Networking & HTTP** — Unit 2's REST/Fetch work is a natural bridge to understanding the full request/response cycle
