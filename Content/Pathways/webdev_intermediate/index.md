# Learning Pathway: Professional Workflow — Intermediate Entry

> ⚠️ **This is a shortcut entry point, not the full journey.**
> If you are starting from zero, follow **[Web Developer: Zero to Employable](../webdev_beginner/index.md)** instead — it covers both courses in the correct order.

> **Who this is for:** Developers who can already build static, interactive HTML/CSS/JS pages but haven't used Git, Promises, or the Fetch API in a real project.
> **Goal:** Add a professional workflow on top of your existing front-end skills.
> **Estimated Time:** ~45 hours
> **Prerequisite:** [Front-End Basics (web)](../../Courses/webdev/index.md) or equivalent hands-on experience with HTML, CSS, and vanilla JavaScript.

## Milestones

**Milestone 1 — "I work with Git and the terminal"**
After Unit 1 (Git & CLI): initialise repos, commit, branch, open pull requests, and deploy via GitHub Pages — all from the command line, no GUI.

**Milestone 2 — "I understand how JavaScript handles time"**
After Unit 2 (Async JS): explain the event loop, write Promises and async/await from memory, and fetch data from a public REST API and render it in the DOM.

**Milestone 3 — "I write modern JavaScript"**
After Unit 3 (ES6+): use destructuring, modules, optional chaining, and expressive array methods as default habits — not things you look up.

**Milestone 4 — "I can persist state and navigate programmatically"**
After Unit 4 (Browser APIs): use `localStorage`, the History API, and the Intersection Observer in real code.

**Milestone 5 — "I can bring UI to life"**
After Unit 5 (CSS Transitions & Animations): animate with `transition`, `transform`, and `@keyframes`, and know when *not* to animate for accessibility.

**Milestone 6 — "I can debug anything"**
After Unit 6 (DevTools) + Capstone: inspect the DOM, read the Network tab, set breakpoints, profile performance — shipped and version-controlled on GitHub.

## Recommended Tools & Environment

- [Git](https://git-scm.com/) installed and configured (`git config --global user.name/email`)
- [Visual Studio Code](https://code.visualstudio.com/) — use the integrated terminal throughout
- A [GitHub](https://github.com) account — you will push real code here
- A modern browser (Chrome or Firefox) — DevTools is used constantly
- A free [Netlify](https://netlify.com) or [GitHub Pages](https://pages.github.com) account for the capstone deployment
