# Foundations of Git

Welcome to **Git Foundations**. This course teaches the version control system that powers modern software development. Whether you're building solo projects, contributing to open-source, or working in a team, Git is the skill you'll use every single day.

## What You'll Learn

In this course, you'll:

- **Understand Git's mental model** — how it stores projects as immutable snapshots in a directed acyclic graph (DAG)
- **Initialize repositories and create commits** — move changes through the three working areas (working directory, staging area, repository)
- **Work with branches** — create isolated feature branches and merge them back together
- **Collaborate with remotes** — push to GitHub, pull changes from teammates, stay in sync
- **Resolve merge conflicts** — handle inevitable situations where two branches modify the same code
- **Apply real workflows** — feature branch workflows, pull requests, and collaborative development

## Prerequisites

- **Comfort with the command line** — You should be able to navigate directories, run commands, and edit files using a terminal
- **A text editor** — VS Code, Sublime, or similar
- **Git installed** — Download from [git-scm.com](https://git-scm.com)
- **A GitHub account** (free) — [Sign up](https://github.com/signup)

No prior version control experience needed. This course builds from zero knowledge.

## Course Structure

### Unit 1: Introduction to Version Control
Understand why version control exists, the three generations of VCS systems, and Git's data model (blobs, trees, commits, DAG). Set the conceptual foundation for everything that follows.

### Unit 2: Initializing a Repository and Making Commits
Initialize your first Git repository, configure identity, stage changes with `git add`, and create commits with clear messages. Master the three working areas.

### Unit 3: Branching and Merging
Create branches to work on features in isolation, understand divergent histories, and merge branches back together with fast-forward and three-way merges.

### Unit 4: Working with Remotes
Push commits to GitHub, pull remote changes, fetch updates, and synchronize between local and remote repositories. Collaborate with teammates.

### Unit 5: Resolving Merge Conflicts
Handle the inevitable: when two branches modify the same lines, Git stops and asks you to decide. Learn to resolve conflicts cleanly and confidently.

## Time Estimate

The full course takes **approximately 15 hours** of active learning and practice:
- Lectures and reading: 6 hours
- Hands-on practice: 7 hours  
- Projects and capstone: 2 hours

You can complete it at your own pace. We recommend spending 2–3 hours per session over a week rather than rushing.

## How to Get the Most Out of This Course

1. **Follow along in your terminal.** Every example is meant to be hands-on. Type the commands yourself — don't just read them.
2. **Experiment in a test repository.** Create a temporary folder, initialize it as a Git repository, and play. Mistakes are free in local repositories.
3. **Reference the knowledge base.** Each unit links to the underlying `.knowledge/` research. Dive deeper if a topic intrigues you.
4. **Build real projects.** By the end, apply Git to a real project — create a GitHub repository, invite teammates to collaborate, or contribute to open-source.

## Learning Outcomes

By the end of this course, you will be able to:

✅ Initialize a Git repository and configure your identity  
✅ Stage and commit changes with clear, meaningful messages  
✅ Create branches for isolated feature development  
✅ Merge branches using fast-forward and three-way strategies  
✅ Push to and pull from GitHub (and other remotes)  
✅ Resolve merge conflicts when they arise  
✅ Use Git confidently in professional team environments  

## Skills Granted

Upon completion, you will have mastered these skills:

- `git:init_repo` — Initialize and configure a Git repository
- `git:add_commit` — Stage changes and create commits
- `git:status_log` — Inspect repository state and commit history
- `git:branch_merge` — Create, switch, and merge branches
- `git:remote_basics` — Configure remotes and sync with GitHub
- `git:resolve_conflicts` — Handle and resolve merge conflicts

These skills are **prerequisites for every advanced Git topic** and for CI/CD, deployment, and collaborative development workflows.
