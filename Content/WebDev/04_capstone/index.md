# Capstone Project — Personal Portfolio Website

> **Project Summary:** You have completed three full units of learning. Now you will prove it. You are being handed a real client brief, wireframes, and a technical specification. Your job is to build the project from scratch — no starter code, no tutorials to follow step by step. Just you, your tools, and the skills you have built.

---

## What This Is

This is not a lesson. There is no "correct" code to copy. The capstone is a **professional simulation** — the kind of assignment a junior developer receives in their first week on the job.

You will receive:

- 📋 **A client brief** — who the client is, what they need, and why
- 🗂️ **User stories** — what the end user should be able to do
- 📐 **Wireframes** — layout reference for desktop and mobile
- ✅ **Acceptance criteria** — the specific checklist your build must pass

---

## What You Will Build

A **single-page personal portfolio website** for a fictional client. The site has six sections:

| # | Section | What it contains |
| :-- | :-- | :-- |
| 1 | Navigation | Logo, nav links, mobile hamburger menu |
| 2 | Hero | Name, tagline, two CTA buttons |
| 3 | About | Photo, bio paragraph |
| 4 | Projects | Three project cards with title, description, links |
| 5 | Skills | Tag-style skill badges |
| 6 | Contact | Form with name, email, message, JS validation |

---

## Deliverables

When you are done, you should have a project folder with exactly:

```
portfolio/
├── index.html
├── style.css
├── app.js
└── README.md
```

No frameworks. No libraries. No CSS resets from CDNs. Everything you need, you already know.
